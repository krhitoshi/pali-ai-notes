# assāsa / passāsa の方向改訂の手順

第一稿の対訳で assāsa / passāsa が逆方向 (assāsa=入息) に訳されている場合に,
プロジェクト方針 (assāsa=出息 / passāsa=入息, 律註定義) へ統一する手順をまとめる.
方向判定の並列スキャンは check_version.md と同じ方式を使う.

## 前提と方針

- CLAUDE.md「翻訳の方針」: assāsa / assasati =「出息 (外へ出る息)」, passāsa /
  passasati =「入息 (内へ入る息)」(律註 Samantapāsādikā の定義を採用)
- 経註・現代慣用や第一稿の一部はこの逆 (assāsa=入息) を採ることがある.
  ファイル内でも日によって方向が揺れる場合がある
- ānāpāna の訳「出入息」「出入息念」は別語であり, assāsa/passāsa の方向とは無関係.
  方向改訂で触らない (むしろ保護対象)

## 全体の流れ

方向だけを一度に機械置換しようとすると破綻する. 次の順に分けて進めると安全:

1. ānāpāna 訳語の統一 (音写除去, 出入息念へ) ― 方向と無関係の前処理
2. 出典・方向注記の削除 (任意) ― 後段の置換対象を減らす
3. 方向判定 ― 各セクションが現在 assāsa=入息 か 出息 かを確定する
4. 方向反転 ― 入息=出息 のセクションだけプレースホルダ方式で反転
5. 派生語の統一 ― 吸気/呼気, 物理説明語 (吸う息/吐く息) を揃える

各段階ごとに別コミットにし, ユーザの動作確認を挟む.

## 1. ānāpāna 訳語の統一 (前処理)

- 音写 (安般 / 安那般那) と逆順 (入出息念) を「出入息念」に統一する
- 「安般」だけを対象にすると全音写形「安那般那」を取りこぼすので両方を見る
- assāsapassāsa 由来の「入出息」はこの段階では触らない (方向バッチで扱う)

## 2. 出典・方向注記の削除 (任意)

- `### 出典*` 見出しブロック, インラインの出典 (`出典：`,`【出典】`,`典拠:`),
  方向注記 (「方向」「該当なし」「assāsa＝入息」系) を削除する
- ブロック内の訳注 (peyyāla 説明, 語義注など) は残す. 出典・方向だけ消す
- 内容説明と方向注記が 1 行に混在する箇所は方向の節だけ外科的に除去する
- 「方向」を含まない注記 (`（参考：…assāsa＝入息）`, `※凡例: …`) も拾う.
  assāsa を含み, かつ慣例・伝統・律蔵註・Visuddhimagga 等の語を含む非 gloss 行が目印

## 3. 方向判定 (最重要)

方向は全セクション一律ではない. 必ずセクション単位で現方向を確定する.

### 機械検出

- 各 `## NNN` をセクション境界とし, gloss から assāsa の訳を読む
- **passāsa は assāsa の部分文字列**である (passāsa ⊃ assāsa, passasati ⊃ assasati).
  正規表現は否定後読み `(?<!p)assāsa` で passāsa を除外する
- **複合語「入出息」内の「出息」を誤検出しない**. 開き括弧直後の息 `「(入息|出息)`
  だけを assāsa の訳とみなす (例 `assāsa-vasena = 「入息…」`).
  「入出息」は `「入息` にも `「出息` にもマッチしないので除外される
- assāsa の gloss が無いセクションは passāsa から逆算する
  (passāsa=出息 → そのセクションは assāsa=入息=要反転)

### 並列サブエージェント (check_version.md の方式)

- 節境界で 5 区間程度に分け, Agent ツールで opus サブエージェントを並列起動する
- 各エージェントは読み取り専用. 各セクションを FLIP / KEEP / NONE に分類し,
  根拠行 (assasati/assāsa/passāsa の gloss, assasissāmi の訳) を挙げさせる
- 報告はそのまま信じず, 機械検出と突き合わせる. 一致しない区画は本文で再確認する

### 判定の確定

- 機械検出と並列スキャンが一致した区画を確定する
- 反転 (FLIP) = assāsa を入息で訳している区画
- 保持 (KEEP) = 既に assāsa=出息 の区画
- 対象外 (NONE) = 方向を持つ息語が無い区画 (反転しても no-op だが, ānāpāna の
  「入息・出息」等を誤反転しないよう範囲から除く)

## 4. 方向反転 (プレースホルダ方式)

FLIP 区画の行範囲だけに次の置換を適用する. 文字単位の単純 swap は複合語を壊すので
プレースホルダで保護してから入れ替える.

```
# 1. 複合語を保護 (ānāpāna 出入息 と assāsapassāsa 入出息 は両方 出入息 へ)
出入息 -> \x00 ; 入出息 -> \x00
# 2. 入息 <-> 出息 (一時プレースホルダ経由)
入息 -> \x01 ; 出息 -> 入息 ; \x01 -> 出息
# 3. 吸気 <-> 呼気
# 3b. 物理説明語 (おくりがなが違うので語単位): 吸う息<->吐く息, 吸いつつ<->吐きつつ, 吸おう<->吐こう
# 4. 復元: \x00 -> 出入息
```

ハマりどころ:

- **出入息 (ānāpāna) は内部に「入息」を含む**. 先に保護しないと 出出息 に壊れる
- **入出息 (assāsapassāsa) は内部に「出息」を含む**. 同上. かつ反転後は 出入息 に
  なるべきなので, ānāpāna の出入息と同じプレースホルダにまとめてよい
- **物理説明語は文字 swap できない**. 吸う (u 動詞) と 吐く (k 動詞) はおくりがなが
  違う (吸おう / 吐こう). 語形ごとに入れ替える
- KEEP 区画内に表記揺れ (assāsa=出息 なのに複合語が「入出息」) が残ることがある.
  複合語統一の観点から個別に 出入息 へ直す

## 5. 派生語の統一

- 一部セクションが「吸気/呼気」語彙で訳している場合, 意味は 呼気=出息 / 吸気=入息 で
  同一なので, CLAUDE.md の表記 (出息/入息) に統一する (呼気→出息, 吸気→入息)
- `呼吸` は別語なので 2 文字単位の置換なら影響しない

## 検証

- 反転後にもう一度方向判定を回し, FLIP / CONFLICT がゼロ (全 KEEP/NONE) を確認する
- 残存「入出息」(assāsapassāsa) がゼロを確認する
- 物理方向の矛盾を grep する (`出息（吸う息）`, `入息（吐く息）` 等が無いこと)
- KEEP 区画 (§163/§165 等) が不変であることを確認する
- diff が対称 swap (挿入数 = 削除数) になっているか目安にする
- 音写の残存 (安般 / 安那般那) がゼロを確認する

## 所見

- 方向は「対訳の内部不整合の目視 → 註/辞書で確定 → 並列スキャンで網羅 →
  反転後の再判定で確認」の順で詰めると確実
- 最高モデルのサブエージェントでも範囲の過大一般化や個別の誤読は出る.
  機械検出との二重化と, 反転後の再判定を必ず行う
- インラインの方向慣例注記 (`〔本系統＝入息〕` 等) が gloss 行に埋め込まれている場合,
  削除バッチで残り反転バッチで向きだけ変わる. 冗長なら別途整理する
